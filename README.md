# RingFileLogger

Универсальный кольцевой файловый логгер для ESP32. Пишет данные в набор файлов фиксированного размера поверх любой файловой системы (`fs::FS`) — при заполнении текущего файла переходит к следующему, а когда заполнены все файлы кольца, самый старый стирается и используется снова.

- Работает поверх любой `fs::FS` — LittleFS, SPIFFS, SD
- Наследуется от `Print` — `print()`, `println()`, `printf()` работают из коробки
- Переживает обрыв питания посреди записи: состояние (какой файл активен, сколько всего было ротаций) восстанавливается из служебных заголовков файлов при следующем `begin()`, без отдельного индексного файла
- Потокобезопасен — все публичные методы защищены внутренним мьютексом, можно писать из разных задач FreeRTOS и с разных ядер одновременно
- Работает с настройками по умолчанию без обязательной конфигурации

## Совместимость

Библиотека совместима с платами на **ESP32** (`framework = arduino`). Внутри используется FreeRTOS-мьютекс, поэтому пока не работает на bare-metal платах без RTOS (AVR, обычные STM32 без FreeRTOS) — такая совместимость запланирована в одной из будущих версий.

Работает поверх любой реализации `fs::FS`: проверено на **LittleFS**, должна без изменений работать и на SPIFFS/SD.

## Установка

**PlatformIO** — добавь в `platformio.ini`:
```ini
lib_deps = Al-Zhilin/RingFileLogger
```

**Arduino IDE** — «Скетч → Подключить библиотеку → Управлять библиотеками...», найти `RingFileLogger`, установить.

**Вручную** — скачать репозиторий и положить в папку `libraries` (Arduino IDE) или `lib/` (PlatformIO).

## Инициализация

```cpp
RingFileLogger logger;   // создаётся без параметров — можно объявлять глобально,
                          // файловая система передаётся позже, в begin()
```

Дальше — `begin()`, две формы:

```cpp
Status begin(fs::FS& filesys);                     // с настройками по умолчанию
Status begin(fs::FS& filesys, const Config& cfg);   // со своими настройками
```

Файловая система (`LittleFS`, `SD`, ...) должна быть уже смонтирована до вызова `begin()` — сама библиотека её не монтирует, только работает поверх. Обе формы возвращают [`Status`](#status) — проверяй на `Status::SUCCESS`.

### Config

```cpp
struct Config {
    uint32_t maxFileSize = 16384;        // максимальный размер одного файла, байт
    uint16_t maxFilesNum = 2;            // количество файлов в кольце
    const char* dirName = "logs";        // директория хранения логов
    const char* filePrefix = "log";      // префикс имени файла
    const char* fileExtension = ".txt";  // расширение файлов
};
```

Ни одно поле не обязательно — если конфиг не передан вовсе, используются значения по умолчанию, указанные выше.

### Status

```cpp
enum class Status {
    SUCCESS,               // всё хорошо
    NOT_INITIALIZED,       // begin() не был вызван или завершился с ошибкой
    FILESYSTEM_ERROR,      // файловая система недоступна
    FILE_OPEN_ERROR,       // не удалось открыть файл
    FILE_CREATE_ERROR,     // не удалось создать файл
    FILE_WRITE_ERROR,      // ошибка записи данных в файл
    INCORRECT_CONFIG,      // в переданном конфиге некорректные значения
    DUMP_OUT_ERROR,        // ошибка вывода во время dumpTo()
    MUTEX_CREATE_ERROR,    // не удалось создать внутренний мьютекс
    MUTEX_TIMEOUT_ERROR,   // не удалось захватить мьютекс за отведённое время
};
```

Возвращается методами `begin()`, `clear()` и `dumpTo()`.

## Использование

### Запись

`RingFileLogger` наследуется от `Print` — как только `begin()` вернул `SUCCESS`, доступны все стандартные методы вывода:

```cpp
logger.println("температура: 23.5");
logger.printf("uptime=%lu\n", millis());
```

В основе лежат два низкоуровневых метода, переопределённых из `Print`:

```cpp
size_t write(uint8_t byte);                     // записать один байт
size_t write(const uint8_t* buf, size_t size);   // записать буфер байт
```

Оба возвращают реальное число записанных байт: `0`, если логгер не инициализирован или данные не влезают даже в пустой файл кольца.

### Обслуживание

```cpp
void flush();     // принудительно сбросить буфер записи на файловую систему
Status clear();   // удалить все файлы кольца и начать заново с "поколения" 1
```

### Чтение истории

```cpp
Status dumpTo(Print& out);
```

Выгружает содержимое всех файлов кольца в переданный `Print` (например, `Serial` или другой `RingFileLogger`) в хронологическом порядке — от самой старой записи к самой новой. Служебные заголовки файлов пропускаются, наружу идут только сами записанные данные.

### Диагностика

```cpp
size_t totalBytesUsed() const;      // суммарный размер всех файлов кольца на ФС, байт
uint32_t currentGenCount() const;   // номер текущего "поколения" — растёт на каждой ротации
uint16_t currentFileNum() const;    // номер файла (0 .. maxFilesNum-1), с которым логгер работает сейчас
```

## Пример

```cpp
#include <Arduino.h>
#include <LittleFS.h>
#include <RingFileLogger.h>

RingFileLogger logger;

void setup() {
    Serial.begin(115200);
    LittleFS.begin(true);

    if (logger.begin(LittleFS) != RingFileLogger::Status::SUCCESS) {
        Serial.println("Ошибка инициализации логгера!");
        return;
    }
}

void loop() {
    logger.printf("millis=%lu\n", millis());
    logger.flush();
    delay(1000);
}
```

Больше примеров, по одному на каждую фичу библиотеки — в папке [`examples/`](examples).
